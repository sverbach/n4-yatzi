# Yatzy — Pass & Play (PWA)

Single-device, pass-and-play Yatzy for up to 6 players.

## Stack

- **Vite 6** — build tooling / dev server
- **React 18 + TypeScript**
- **vite-plugin-pwa** (Workbox `generateSW`) — installable, offline-capable PWA
- **Tailwind CSS v4** — styling, with design tokens exposed as a themeable palette
- **Zustand** — game state

## Design tokens & theming

The handoff's bound design system (`_ds/…`, Solarized-derived) wasn't bundled, so
its tokens are recreated in [`src/index.css`](src/index.css) as CSS custom
properties. Light is the default; the dark palette re-resolves the same variable
names under `[data-theme="dark"]`, and the theme attribute is toggled on the root
element from the store's `darkMode` flag. Tokens are mapped into Tailwind via
`@theme inline` so utilities like `bg-surface` / `text-ink` / `border-rule-soft`
stay live-bound to the variables and flip with the theme.

## Structure

```
src/
  game/scoring.ts        # categories, calc formulas, totals, pip layout
  store/gameStore.ts     # Zustand store: phases, dice/roll/hold/revert, scoring
  components/
    SetupScreen.tsx      # phase: 'setup'
    PlayScreen.tsx        #   header + Scoresheet + DiceArea
    Scoresheet.tsx        #   6-seat grid, live score previews
    DiceArea.tsx / Die.tsx
    ResultsScreen.tsx    # phase: 'finished'
    ui/                  # Button, Card, Input, Badge, Divider, Tooltip
```

## Scripts

```bash
npm install
npm run dev        # dev server
npm run build      # typecheck + production build (emits service worker)
npm run preview    # serve the production build (PWA/service worker active)
```

Icons are generated by `node scripts/generate-icons.mjs` (self-contained PNG
encoder — no image dependencies).

## Notes

- **Persistence**: game state is saved to `localStorage` via Zustand's `persist`
  middleware (key `yatzy-game`), so a game — including the current turn's dice,
  rolls-left, and the dark-mode preference — survives reloads and app relaunch.
  Transient fields are excluded: the in-flight `newPlayerName` input and the
  timer-driven `rolling` flag (which resets to `false` on rehydrate, leaving the
  settled dice intact).
